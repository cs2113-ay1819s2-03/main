PROJECT: NoteNote

Overview

NoteNote is an effective tool for organizing multiple projects and tasks through the Command Line Interface (CLI).

Other than keeping track of your projects and tasks, NoteNote allows you to analyze your productivity, categorize your tasks, share tasks with you friends and many more.

Import a project list: import

Use this function when importing a project list shared by a friend to import additional projects and tasks on top of your current project list.

Source file must be a JSON file and follow specific format. See [samplejson].

This feature is recommended to be used together with export so that users need not manually generate a JSON file to import.

  • Format: import PATH

  • Alias: i, i.e. import is equivalent to i

Examples:

  • import C:\Users\Test\Desktop\import.json

Figure 2. Before executing import
Figure 3. After executing import
The full path to the JSON file has to be specified unless the file is contained within the application folder. If JSON file is contained in application folder, import .\test.json is sufficient.

Outcome: Projects and tasks contained in JSON file will be imported and added on top of current project list. Command box will display number of projects added to project list.

Export projects to JSON file: export

You could export projects from your NoteNote instance to easily share project tasks with your friends. Projects are exported in JSON format and could be imported by another NoteNote instance using import.

  • Format: export i/INDEX o/OUTPUTPATH

  • Alias: ex, i.e. export is equivalent to ex

Examples:

  • To export multiple projects: export i/1,2,3 o/C:\Users\Test\Desktop\export.json

  • To export currently selected project: export o/C:\Users\Test\Desktop\export.json

Output path should be explicitly specified unless the output is to be within the application folder. If output is to be contained in application folder, the export i/1,2,3 o/.\test.json is sufficient.

Outcome: JSON file will be created at output path specified.

Import/Export feature

Current Implementation

The import feature reads projects from a JSON file provided as input and adds these projects on top of the current list of projects. The mechanism is facilitated by JsonUtil and is achieved using the following functions:

  • JsonUtil#readJsonFile() — Read projects from JSON file

  • VersionedProjectList#addProject() — Adds projects from JSON file to project list.

The export feature exports the current selected project (after select 1) or projects specified by their index to a JSON file. The mechanism is facilitated by FileUtil and JsonUtil and is achieved using the following functions:

  • FileUtil#createIfMissing() — Creates JSON file to write to if it does not exist

  • JsonUtil#saveJsonFile() — Writes projects to JSON file

Given below is an example usage scenario and how the import/export mechanism behaves at each step.

Step 1. The user launches the application, projectList will be populated by invoking StorageManager#readProjectList.

If user launches the application for the first time, projectList will be populated with sample project list as ./data/projectlist.json does not exist.

Step 2. The user executes import ./data/import.json to add projects from JSON file to versionedProjectList.

  1. Path to JSON file is passed to JsonUtil#readJsonFile() which would read projects to a temporary project list.

  2. Application will then loop through all projects and add them to versionedProjectList using VersionedProjectList#addProject().

  3. updateFilteredProjectList is then executed to refresh ProjectListPanel with all projects, including recently imported projects.

Step 3. The user executes export i/1,2,3 o/./data/export.json to export projects with index 1, 2 and 3.

  1. Application will get the projects from filteredProjectList and add them to a new ProjectList projectsToExport.

  2. projectsToExport is then passed to JsonUtil#saveJsonFile() which would write to output file specified by user.

The entire project list could be found in ./data/projectlist.json

Method of tracking multiple projects within project list and multiple tasks within each project

Each project will have its own <Project File>.json to keep track of tasks.

  • Pros: Easy to implement.

  • Cons: May clutter data folder with too many project files. Application will have to read a new project file whenever user switches project.

Current Choice

All projects within project list and all tasks of all projects contained in project.json file.

  • Pros: Will not clutter data folder.

  • Cons: Entire project list has to be written to file every time project list is modified.

This option was chosen as it facilitates importing and exporting projects to one file only.
